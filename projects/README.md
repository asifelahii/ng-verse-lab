# Angular Practice Projects (Resume-ready)

These are NOT random demos. Each project is designed to cover Angular core pillars + interview topics.

## Project List

### P01 — MiniMart Catalog (Fundamentals)

**Folder:** `projects/01-fundamentals/`  
**Goal:** Components + templates + binding + directives + pipes (clean UI, no backend).  
**Pillars covered:** Component architecture, Data binding, Directives, Pipes, Basic TS.

**Deliverables**

- List UI with filtering/search
- Reusable components (Card, Badge)
- Custom pipe(s)
- Clean folder structure inside the Angular app

---

### P02 — Student CRUD (Forms + Validation)

**Folder:** `projects/02-forms-crud/`  
**Goal:** Reactive forms, validation, CRUD in memory/localStorage.  
**Pillars covered:** Forms management, State handling (local), Component interaction.

**Deliverables**

- Create/Edit/Delete
- Reactive forms + validators
- Simple state service

---

### P03 — RxShop (HTTP + RxJS)

**Folder:** `projects/03-http-rxjs/`  
**Goal:** Real API calls + RxJS streams (loading/error/retry/caching).  
**Pillars covered:** Services & HTTP, RxJS, State handling (service/store-lite).

**Deliverables**

- Products fetched from API
- Search stream + debounce
- Caching + error handling

---

### P04 — RouteShield (Routing + Auth)

**Folder:** `projects/04-routing-auth/`  
**Goal:** Routing, guards, lazy loading, auth simulation.  
**Pillars covered:** Routing/navigation, DI, State (auth), Architecture.

**Deliverables**

- Login/logout (mock)
- Auth guard + protected routes
- Lazy loaded feature module(s)

---

### P05 — TestBench (Unit + Integration + E2E basics)

**Folder:** `projects/05-testing/`  
**Goal:** Testing mindset, not just syntax.  
**Pillars covered:** Testing, DI mocking, Component testing, HTTP testing.

**Deliverables**

- Unit tests for components/services
- Router testing
- HTTP mocks
